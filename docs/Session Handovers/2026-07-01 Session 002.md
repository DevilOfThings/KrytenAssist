# Development Workflow

For every prompt

1. Discuss the design.
2. Agree the architecture.
3. Create the AI Prompt.
4. Implement one file at a time.
5. Review generated code.
6. Build.
7. Test.
8. Update AI Playbook.
9. Commit to Git.
10. Update Session Handover.
11. Review Roadmap.

---

# Lessons Learnt

- One prompt may describe a complete feature, but implementation should remain one file at a time.
- Building after every file dramatically reduces debugging effort.
- Every prompt should finish with:
    - Successful build
    - Swagger verification (where applicable)
    - AI Playbook update
    - Git commit
- AI prompts should include:
    - Current architecture
    - Existing namespaces
    - Method signatures
    - Repository contracts
    - Existing endpoints
    - Success criteria
- AI prompts have evolved into design specifications rather than simple coding instructions.
