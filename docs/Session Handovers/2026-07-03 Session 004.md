# Completed During This Session

## Prompt 014 – Integration Tests ✅

### Goal

Introduce automated API integration tests using `WebApplicationFactory` to verify the PromptCard API through the complete HTTP pipeline.

### Completed

Created a dedicated integration test project and established a maintainable testing structure organised by feature.

### Added

```text
KrytenAssist.Api.Tests/
│
├── Infrastructure/
│   └── CustomWebApplicationFactory.cs
│
├── PromptCards/
│   ├── CreatePromptCardTests.cs
│   ├── GetPromptCardTests.cs
│   ├── UpdatePromptCardTests.cs
│   └── DeletePromptCardTests.cs
│
└── TestData/
    └── PromptCardRequests.cs
```

### Program Updates

`Program.cs` now exposes:

```csharp
public partial class Program;
```

allowing `WebApplicationFactory<Program>` to host the API during integration testing.

### Test Coverage

Implemented automated tests covering:

- GET all PromptCards
- GET existing PromptCard
- GET unknown PromptCard
- POST valid request
- POST invalid request
- PUT existing PromptCard
- PUT unknown PromptCard
- DELETE existing PromptCard
- DELETE unknown PromptCard

Total:

```text
9 Integration Tests
9 Passing
0 Failing
```

### Design Decisions

- Tests interact only through `HttpClient`.
- No repositories or use cases are called directly.
- Tests are grouped by feature rather than endpoint.
- Shared request creation is centralised in `PromptCardRequests`.
- Each test is independent and creates its own data.
- The API is exercised end-to-end through the full ASP.NET Core pipeline.

2. Agree the architecture.
3. Create the AI Playbook prompt.
4. Implement one file at a time.
5. Build after each significant step.

# Lessons Learnt

- Organising tests by feature scales far better than a single endpoint test class.
- Shared test data builders reduce duplication and simplify maintenance.
- Integration tests should validate behaviour through the complete HTTP pipeline.
- Keeping prompts focused on one architectural concern results in cleaner commits and easier reviews.
- The `dns` shorthand proved an efficient workflow for stepping through prompts.
- AI Playbook prompts should be updated after implementation to reflect the final design, not just the original plan.
