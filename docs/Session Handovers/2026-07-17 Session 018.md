## Architecture

- Core owns stable sailing identity, meaningful-change fingerprints, comparable
  price selection, summaries and trend analysis.
- Application owns `ICruiseObservationRepository`, immutable recorded-history
  results and record/get/list use cases.
- Infrastructure owns EF Core entities, SQLite mapping, migrations,
  transactions, concurrency handling and repository implementation.
- Avalonia owns desktop composition, commands, cancellation generations and
  local list/detail presentation.
- Views remain passive, and native-browser code-behind contains no persistence
  behavior.
- Infrastructure is referenced from Avalonia only through the established
  composition alias.

## Identity and Source Rules

One physical sailing is identified by normalized:

- operator id
- ship name
- departure date
- duration

Retail source is deliberately separate. The same sailing advertised by TUI and
another retailer has independent recorded histories and price series.

Provider offer/package id, source reference, observed price, promotion and
timestamp remain evidence; they are not the sole physical sailing identity.

## Meaningful Changes and Latest Evidence

The fingerprint includes canonical advertised offer facts, prices, promotion
and source identity. Equivalent case/whitespace, price ordering and duplicate
price representations do not create false changes.

- First evidence creates one history and one snapshot.
- Meaningful advertised change creates a new chronological snapshot.
- Identical current evidence creates no duplicate snapshot.
- Last-seen and latest offer/reference evidence may advance independently.
- Returning to a prior advertised state after an intervening state creates a new
  chronological snapshot.
- Equal timestamps use deterministic fingerprint/evidence ordering.

## Price History

Kryten displays:

- first observed
- last observed
- last seen
- current comparable price
- lowest comparable price
- highest comparable price
- recorded observation count
- First, Up, Down, Unchanged or Unavailable trend
- latest offer id and source reference

Price comparison never mixes currency or basis. Ambiguous prices remain
unavailable rather than guessed. The original observation prices remain stored
even when no headline comparison can be calculated.

## SQLite Persistence

The existing `KrytenAssistDbContext` now includes normalized:

- `CruiseHistories`
- `CruiseObservations`
- `CruiseObservationPrices`

Checked-in migrations:

- `20260717082520_AddCruiseHistoryPersistence`
- `20260717090431_HardenCruiseHistoryRecording`

The schema enforces sailing/source uniqueness, positive observation sequence,
ordered price uniqueness, foreign keys, cascades and bounded values. Prompt Card
data remains in the same database and survives migration.

Recording uses an explicit transaction. Tests prove rollback on cancellation or
constraint failure, concurrent first/changed recording, distinct concurrent
changes, migration from earlier schemas and complete disposal/recreation.

The desktop database is stored beneath the platform Local Application Data path
at `KrytenAssist/krytenassist.db`. Automated tests override this path and never
touch Robin's real database.

## Presentation and Lifecycle

- Cruise History loads locally when Cruise of the Week is selected.
- This activation performs no TUI, browser, HTTP or Skill execution.
- Record, cancel, refresh and cancel-loading commands expose controlled states.
- Successful recording refreshes and selects the affected sailing/source.
- Failed refresh retains the last successfully loaded list.
- Cancellation and capture replacement reject stale asynchronous results and
  permit immediate retry.
- Browser navigation, refresh and close clear only capture-review state; loaded
  and persisted history remains intact.

## Known Limitations

The currently captured known price is stored and displayed neutrally. The domain
does not yet distinguish:

- original price
- discounted price
- per-person discount
- an extra booking-level discount such as £100 off the total booking

That model should be introduced explicitly rather than inferred from promotion
text.

Prompt 037 also contains no ratings, interest, notes, favourites, preference
profile, recommendation, cross-retailer ranking, alerting or booking behavior.

The existing SQLite package advisory remains a separate dependency-maintenance
task.
