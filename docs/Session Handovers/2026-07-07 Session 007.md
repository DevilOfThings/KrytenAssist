## Lessons Learnt

### React Component Lifecycle

A subtle bug caused the success message to disappear immediately after creating a Prompt Card.

Root cause:

`PromptCardList` returned early while loading, temporarily removing `PromptCardForm` from the component tree.

This caused React to unmount the component, clearing all local state including the success message.

Instead of:

- replacing the whole page during loading,

the solution was to:

- keep the editor mounted
- only display loading for the browser section

This preserves component state and provides a much better user experience.

**Key principle**

> Avoid early returns that unmount unrelated UI during loading or error states. Keep stable components mounted whenever possible.
