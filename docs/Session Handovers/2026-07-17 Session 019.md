# Session Handover 019 – Prompt 037h Multiple Cruise Deals Handling

**Date:** 17 July 2026

## Session Summary

Prompt 037h is complete.

Kryten can explicitly capture up to ten currently loaded Cruise deals from a
supported TUI product-card page, review independent provider-neutral
observations, and explicitly record selected or all Ready observations into
local Cruise History.

## Completed Workflow

```text
Open supported TUI deal page
        ↓ Robin chooses Capture Loaded Cruises
Independently validate up to ten loaded cards
        ↓
Review Ready / Incomplete / Failed candidates
        ↓ Robin chooses Record Selected or Record All Observations
Sequential controlled per-row outcomes
        ↓
One local Cruise History refresh after useful results
```

Capture and recording remain explicit. Kryten does not load more cards, change
filters, record automatically, scrape in the background, recommend cruises or
perform booking activity.

## Safety and Architecture

- The Application batch contract remains provider independent.
- The TUI adapter scopes evidence to one demonstrated
  `[data-testid="product-card"]` at a time inside `tui-product-cards`.
- Each Ready observation retains its own validated itinerary reference.
- Exact duplicate links deduplicate in first-seen order; at most ten currently
  loaded candidates cross the boundary and truncation is visible.
- Avalonia orchestrates a stable sequential snapshot through the existing
  `RecordCruiseObservation` use case; it does not write directly to SQLite.
- Each row retains First, Changed, Already Current, Failed or Cancelled state.
- Useful batch results refresh History once; failures do not roll back other
  sailings.

## Known Limits and Follow-ups

- TUI Italy destination pages use a distinct `small-product-card` template.
  Kryten intentionally reports this as unsupported rather than guessing across
  card boundaries. Add support only through a later narrowly scoped prompt with
  fixed fixtures and isolation tests.
- The multi-Cruise review layout makes it difficult to navigate sources while
  checking incompatible pages. Plan a separate presentation refinement before
  broadening TUI template support.
- Prompt 038 remains unstarted: Saved Cruises and Preferences.
- Original price, discounted price, per-person discount and booking-level
  discount remain a separate future price-model change.

## Recommended Next Task

source support, plan a small Cruise workspace presentation prompt so source
navigation and batch review remain comfortable. Treat alternate TUI card
template support as a separate source-adapter prompt, not as a selector
