## Architecture

- Core owns immutable, provider-independent Cruise data.
- Application owns the bounded capture request, result statuses and capture
  service interface.
- Infrastructure owns the private TUI version-1 payload and its mapping into
  Cruise models.
- Avalonia owns trusted source navigation, the native browser, the fixed TUI
  script, capture state and the review presentation.
- Infrastructure is referenced from Avalonia only through composition aliases.

Marella Cruises remains the cruise operator. TUI is retained separately as the
retail or advertising source.

## Browser and Security Boundary

- Only exact HTTPS addresses on the configured TUI host are trusted.
- `about:blank` is treated only as browser preparation.
- Deceptive suffixes and unrelated hosts are rejected.
- Capture uses one fixed, bounded, read-only TUI script.
- The script does not click, submit, scroll, mutate the DOM, read cookies or
  browser storage, or return full HTML.
- Candidate results, strings, semantic scans and supported roots are bounded.
- Cancellation generations reject late results after navigation, Refresh,
  Cancel or Close.
- External opening is explicit and exact-address validated.

## TUI Page Shape

The live Marella Cruise of the Week offer is rendered inside the open shadow
root of TUI's `tui-product-cards` web component. Ordinary top-level document
selectors cannot see that offer.

Kryten explicitly reads only that demonstrated component and deduplicates the
two identical itinerary links found there. This is intentionally not a generic
recursive shadow-DOM crawler.

TUI may change this page shape in future. A controlled Incomplete or Failed
result is preferable to invented Cruise data; update the source-specific adapter
only after a new live page shape is demonstrated.
