# Problems Encountered

### EF Core Package Version Conflict

A package downgrade warning (`NU1605`) occurred due to a mismatch in `Microsoft.Extensions.DependencyInjection` versions.

Resolution:

- Updated Infrastructure package references to use matching 10.0.9 versions.

---

### SQLite DateTimeOffset Ordering

Entity Framework Core cannot translate `DateTimeOffset` directly into SQLite `ORDER BY` clauses.

Resolution:

- Added `ValueConverter` to persist `CreatedAt` and `UpdatedAt` as ISO-8601 strings.
- Preserved correct chronological ordering while maintaining the domain model.

---

### Tags Change Tracking

Entity Framework Core warned that collection properties using a `ValueConverter` require a `ValueComparer`.

Resolution:

- Added a `ValueComparer` for `Tags` to ensure correct change tracking.

---

### Repository Database

The generated SQLite database file (`krytenassist.db`) was accidentally tracked by Git.

Resolution:

- Removed it from source control.
- Added it to `.gitignore`.

# Lessons Learnt

- Repository abstractions allowed a complete persistence implementation swap without changing the API.
- Entity Framework Core significantly reduced repository complexity.
- SQLite requires `DateTimeOffset` values to be converted when ordering.
- Collection properties using `ValueConverter` should also define a `ValueComparer`.
- Integration tests provided confidence that the API contract remained unchanged throughout the migration.
- Removing generated database files from source control keeps the repository clean and reproducible.
