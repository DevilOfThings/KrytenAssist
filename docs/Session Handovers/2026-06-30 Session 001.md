# Development Workflow

1. Discuss the design.
2. Agree the architecture and responsibilities.
3. Write the AI Playbook prompt.
4. Create the target file(s).
5. Open the target file.
6. Ask Kryten to implement the currently open file.
7. Review the generated code.
8. Build the solution.
9. Update the AI Playbook:
    - Result
    - Notes / Lessons Learnt
10. Commit to Git.
11. Review the roadmap and agree the next vertical slice.

---

# Lessons Learnt

- One AI prompt can describe an entire feature, but implementation should target one file at a time.
- Building after every vertical slice makes problems easier to isolate.
- Every completed prompt should end with:
    - Successful build
    - AI Playbook updated
    - Git commit
- Session handovers provide a reliable restart point and should be stored in source control rather than relying on conversational memory.
